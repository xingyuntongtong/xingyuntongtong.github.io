    3. mongodb启动与连接

    3.1 windows启动服务器端

    1.找到mongodb安装目录,一般是 C:\Program Files\MongoDB 2.6 Standard\bin
    2.按下Shift+鼠标右键,选择在此处打开命令窗口(就是DOM命令窗口)
    3.在除C盘外的盘符新建一个空目录,如 D:\Mongodb\data
    4.在打开的命令行中输入mongod –dbpath=刚创建的空目录,如
    mongod –dbpath=D:\Mongodb\data 再按回车键
    5.如果出现waiting for connections on port 27017就表示启动成功,已经在27017端口上监听了客户端的请求
    6.注意：–dbpath后的值表示数据库文件的存储路径,而且后面的路径必须事先创建好，必须已经存在，否则服务开启失败，每个操作数据库的时候都得启动服务器
    7.注意：这个命令窗体绝对不能关,关闭这个窗口就相当于停止了mongodb服务

    3.2 启动客户端连接服务器

    1.找到mongodb安装目录,一般是 C:\Program Files\MongoDB 2.6 Standard\bin
    2.按下Shift+鼠标右键,选择在此处打开命令窗口 ，就是和上面的路径一样，在重新打开一个
    3.命令窗体中输入 mongo –host=127.0.0.1 或者 mongo 按回车键

    备注：–host后的值表示服务器的ip地址 备注： –host=127.0.0.1 表示的就是本地服务器,每次数据库都会默认连接test数据库

    4. MongoDB基本概念

  • 数据库 MongoDB的单个实例可以容纳多个独立的数据库,比如一个学生管理系统就可以对应一个数据库实例 Database
  • 集合 数据库是由集合组成的,一个集合用来表示一个实体,如学生集合 Collections:内容都在这里
  • 文档 集合是由文档组成的，一个文档表示一条记录,比如一位同学张三就是一个文档,也就是里面的内容

5.1 主键

当我们保存一个文档的时候，如果没有提供_id属性，mongodb会帮我们自动生成一个_id.这就是主键
1.它与业务无关
2.每个文档的主键不能相同，也不能为空，就像身份证号一样
3.无论文档的属性如何改变，主键不能修改，就像年龄一样，怎么变这个身份证号也不会变
4.只要知道了这个主键，就能找到其它的属性
5.主键能唯一标识一个文档

7. 更新文档

  • query(_id:1) 查询条件,指定要更新符合哪些条件的文档
  • upsert 可选，这个参数的意思是，如果不存在符合条件的记录时是否插入updateObj. 默认是false,不插入。
    如果upsert=ture代表找到就修改，没有就插入
  • multi 可选，mongodb 默认只更新找到的第一条记录，如果这个参数为true,就更新所有符合条件的记录。
